# README: Дашборд энергопотребления

## Назначение

Дашборд предназначен для просмотра и анализа исторических данных энергопотребления по счетчикам, помещениям и классам потребителей. Интерфейс работает с подготовленными данными из backend API и отображает агрегированную картину потребления за выбранный период.

Основной экран находится в:

```text
src/pages/DashboardPage.jsx
```

Стили дашборда находятся в:

```text
src/dashboard.css
```

API-клиент дашборда находится в:

```text
src/features/dashboard/api/dashboardApi.js
```

## Источник данных

Фронтенд получает данные через API:

```text
/api/consumption/dashboard/
```

Используемые endpoints:

```text
GET /api/consumption/dashboard/filters/
GET /api/consumption/dashboard/summary/
GET /api/consumption/dashboard/timeseries/
GET /api/consumption/dashboard/top-devices/
GET /api/consumption/dashboard/room-loads/
GET /api/consumption/dashboard/devices/<data_name>/
```

Основные данные потребления берутся из таблицы `power_1min`, где хранятся минутные измерения. Справочная информация по устройствам берется из таблицы `devices`, а связи с помещениями и потребителями используются из таблиц `breakers` и `consumers`.

## Основной Функционал

Дашборд реализует:

- загрузку реальных счетчиков, помещений и классов потребителей;
- фильтрацию данных по счетчику;
- фильтрацию данных по помещению;
- фильтрацию данных по классу потребителя;
- выбор периода анализа;
- построение линейного графика активной мощности;
- интерактивную подсказку на графике при наведении;
- расчет KPI по выбранной выборке;
- отображение топа счетчиков по энергии;
- отображение потребления по помещениям;
- краткую аналитическую сводку;
- просмотр связей выбранного счетчика с потребителями и автоматами.

## Фильтры

### Счетчик

Фильтр `Счетчик` позволяет выбрать конкретный прибор учета. Если выбран конкретный счетчик, фильтры по помещению и классу потребителя сбрасываются, чтобы не смешивать несовместимые условия.

Значение `Все счетчики` означает, что расчеты выполняются по всем доступным счетчикам в выбранной выборке.

### Помещение

Фильтр `Помещение` позволяет построить показатели только для счетчиков, связанных с выбранным помещением через справочники `consumers` и `breakers`.

При выборе помещения счетчик сбрасывается в режим `Все счетчики`.

### Класс

Фильтр `Класс` позволяет смотреть потребление по классу потребителей, например освещение или другая категория нагрузки.

При выборе класса счетчик сбрасывается в режим `Все счетчики`.

### Период

Доступны режимы:

```text
Весь период
24 часа
7 дней
```

Для режима `Весь период` фронтенд не передает параметры `from` и `to`, поэтому backend возвращает данные за весь доступный диапазон базы.

Для режима `24 часа` используется выбранная дата:

```text
from = YYYY-MM-DDT00:00:00Z
to   = YYYY-MM-DDT23:59:59Z
```

Для режима `7 дней` выбранная дата считается последним днем периода. Начало периода рассчитывается как выбранная дата минус 6 дней. Если рассчитанная дата выходит за нижнюю границу базы, она ограничивается минимальной доступной датой.

## Линейный График

График `Динамика активной мощности` отображает активную мощность по выбранной группе счетчиков.

Используемая метрика:

```text
active_power_w_avg
```

Backend агрегирует данные по времени. Для активной мощности используется сумма значений по выбранным счетчикам на каждый момент времени.

Фронтенд переводит значения из ватт в киловатты:

```text
кВт = Вт / 1000
```

### Интерактивность

График поддерживает наведение курсора. При наведении отображается подсказка с:

- датой и временем точки;
- значением активной мощности в кВт.

Также на графике появляется вертикальный маркер и выделяется ближайшая точка линии.

### Средняя Линия

Пунктирная линия показывает среднюю мощность по отображаемым точкам графика.

Расчет:

```text
Средняя мощность = сумма всех значений мощности / количество точек
```

### Пики

Точки, которые заметно выше среднего уровня, подсвечиваются как пики. Сейчас пик на графике считается точкой, значение которой не меньше `125%` от среднего значения отображаемой линии.

```text
Порог пика = Средняя мощность * 1.25
```

## KPI

В верхней части дашборда отображаются ключевые показатели выбранной выборки.

### Средняя мощность

Показывает среднюю активную мощность за выбранный период по агрегированной временной серии.

Расчет:

```text
Средняя мощность = сумма значений линии графика / количество точек линии графика
```

Единица измерения:

```text
кВт
```

Этот показатель рассчитывается на фронтенде по тем же данным, по которым строится линейный график.

### Энергия за период

Показывает суммарную расчетную энергию за выбранный период.

Источник:

```text
summary.total_energy_kwh
```

Backend считает ее как сумму поля:

```text
energy_kwh_est
```

Единица измерения:

```text
кВт·ч
```

### Максимальная мощность

Показывает максимум агрегированной активной мощности за выбранный период.

Расчет:

```text
Максимальная мощность = max(значения линии графика)
```

Единица измерения:

```text
кВт
```

Важно: показатель считается по агрегированной линии графика, а не по максимуму отдельного счетчика. Это нужно, чтобы значение соответствовало выбранной группе и визуализации на графике.

В подписи карточки отображается время пика.

### Счетчики в выборке

Показывает количество уникальных счетчиков, попавших в текущую выборку.

Источник:

```text
summary.devices_count
```

В подписи карточки отображается количество точек периода, использованных для графика.

## Топ Счетчиков

Блок `Топ счетчиков` показывает счетчики с наибольшим суммарным потреблением за выбранный период.

Для каждого счетчика отображаются:

- позиция в рейтинге;
- название счетчика;
- суммарная энергия в кВт·ч;
- максимальная мощность счетчика в кВт.

Сортировка выполняется по суммарной энергии:

```text
energy_kwh DESC
```

## Потребление По Помещениям

Блок `Потребление по помещениям` показывает распределение энергии между помещениями, связанными со счетчиками через справочники.

Для каждого помещения отображаются:

- номер или название помещения;
- горизонтальная шкала относительного потребления;
- энергия в кВт·ч.

Длина шкалы рассчитывается относительно помещения с максимальным потреблением в текущей выборке.

## Краткая Сводка

Блок `Краткая сводка` содержит дополнительные показатели, полезные для анализа выбранной выборки.

### Средняя мощность периода

Дублирует среднюю мощность по агрегированной линии графика, чтобы показатель был доступен рядом с остальными аналитическими характеристиками.

Расчет:

```text
Средняя мощность периода = avg(значения линии графика)
```

### Минимальная мощность

Показывает минимальную активную мощность за выбранный период.

Расчет:

```text
Минимальная мощность = min(значения линии графика)
```

### Пик нагрузки

Показывает максимальную активную мощность и дату/время, когда она была достигнута.

Расчет:

```text
Пик нагрузки = max(значения линии графика)
```

### Коэффициент загрузки

Показывает, насколько равномерной была нагрузка относительно максимума.

Расчет:

```text
Коэффициент загрузки = Средняя мощность / Максимальная мощность * 100%
```

Интерпретация:

- высокий процент означает более ровное потребление;
- низкий процент означает, что были выраженные пики относительно среднего уровня.

### Среднее напряжение

Показывает среднее напряжение по доступным фазам счетчиков.

Источник:

```text
summary.avg_voltage_v
```

Backend считает среднее по полю:

```text
voltage_avg_v
```

Единица измерения:

```text
В
```

### Средняя частота

Показывает среднюю частоту сети в выбранной выборке.

Источник:

```text
summary.avg_frequency_hz
```

Backend считает среднее по полю:

```text
frequency_hz_avg
```

Единица измерения:

```text
Гц
```

### Самый Энергоемкий Счетчик

Показывает счетчик с максимальным суммарным потреблением за выбранный период.

Источник:

```text
topDevices[0]
```

Показатель полезен для быстрого определения главного потребителя энергии в текущей выборке.

### Диапазон Выборки

Показывает фактический временной диапазон, за который backend вернул данные после применения фильтров.

Источник:

```text
summary.date_from
summary.date_to
```

### Строк Измерений

Показывает количество сырых минутных записей, попавших в текущую выборку.

Источник:

```text
summary.points
```

Важно: это не то же самое, что количество точек графика. Если выбрано несколько счетчиков, в один момент времени может быть несколько сырых строк, но на графике они агрегируются в одну точку.

## Вкладка Связи

Вкладка `Связи` показывает справочную информацию по выбранному счетчику.

Чтобы увидеть данные, нужно выбрать конкретный счетчик.

Отображаются два блока:

### Потребители

Показывает записи из справочника потребителей:

- потребитель;
- класс потребителя;
- помещение.

### Автоматы И Помещения

Показывает связи счетчика с автоматами:

- автомат;
- помещение;
- этаж и корпус.

## Обработка Ошибок И Состояний

Если API возвращает ошибку, сообщение отображается в верхней части дашборда.

Если данные загружаются, в боковой панели отображается статус загрузки.

Если для выбранных фильтров нет данных, график показывает сообщение:

```text
Нет данных для выбранных фильтров
```

## Особенности Расчетов

Мощностные KPI рассчитываются по временной серии, а не напрямую по `summary`, потому что линия графика уже является агрегированной суммой по выбранной группе. Это делает KPI визуально и аналитически согласованными с графиком.

Энергия, количество счетчиков, среднее напряжение, средняя частота и количество сырых строк берутся из `summary`, потому что эти показатели корректно считаются backend-запросом по исходным данным.

## Ограничения

- Данные являются историческими, а не live-телеметрией.
- Доступный диапазон дат зависит от содержимого базы.
- Для периода `7 дней` начало периода ограничивается нижней границей доступных данных.
- График отображает ограниченное количество точек путем сэмплирования, чтобы интерфейс оставался быстрым и читаемым.
- Tooltip на графике показывает ближайшую отображаемую точку, а не обязательно каждую сырую минутную запись.
