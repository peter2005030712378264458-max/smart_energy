# Веб-приложение для мониторинга энергопотребления

## Описание проекта

Данный проект представляет собой веб-приложение для мониторинга и анализа энергопотребления на объектах.
Приложение берет данные из базы данных и предоставляет инструменты для визуализации этих данных.

Основная цель системы — обеспечить удобный доступ к информации об энергопотреблении и помочь выявлять зоны с повышенным, пониженным или аномальным потреблением энергии.

---

## Основной функционал

### 1. Дашборд

Приложение включает дашборд, построенный на основе базы данных, содержащей данные с датчиков энергопотребления, установленных в различных помещениях объекта.

Дашборд позволяет:

* просматривать текущее и историческое энергопотребление;
* сравнивать энергопотребление между различными помещениями или этажами;
* отображать данные в виде графиков и других визуализаций.
* выбирать параметры визуализации (что визуализировать (агрегации) и как). Это находится на отдельной странице дашборда

Дашборд может состоять из нескольких страниц. Страницы могут быть структурированы:

* по **этажам** объекта;
* по **помещениям**;
* по **типам графиков**;
* по **типам отображаемой информации**.

Пока точного плана структуризации нет

---

### 2. Карта объектов

В приложении также будет реализована интерактивная карта объектов.

На первом этапе система будет поддерживать **один объект**, однако архитектура будет предусматривать возможность добавления **нескольких объектов в будущем**.

Функциональность карты включает:

* отображение помещений объекта;
* отображение датчиков на карте;
* возможность выбора конкретного датчика;
* просмотр данных выбранного датчика в виде визуализаций (в том числе с историческими данными).

---

### 3. Режим тепловой карты

Дополнительно будет реализован режим **тепловой карты (heatmap)**.

В этом режиме пользователи смогут визуально определить уровень энергопотребления в разных помещениях объекта.

Тепловая карта позволит:

* быстро определить помещения с **высоким энергопотреблением**;
* увидеть помещения с **низким энергопотреблением**;
* сравнивать распределение энергопотребления по объекту.

Цветовая визуализация будет отображать уровень энергопотребления, что позволит быстрее анализировать данные.

### 4. Личный кабинет пользователя

В системе также будет реализован **личный кабинет пользователя**.

Каждый пользователь сможет иметь собственную учетную запись, через которую он будет получать доступ к функционалу системы, включая дашборды, карты объектов и визуализацию данных датчиков.

Создание новых пользователей будет происходить **только с разрешения администратора**.
Администратор — это назначенное ответственное лицо, которое управляет доступом к системе.

Функциональность личного кабинета может включать:

* авторизацию пользователя в системе;
* доступ к данным в соответствии с назначенными правами;
* просмотр доступных объектов и датчиков;
* работу с дашбордами и визуализациями.

Администратор системы будет иметь дополнительные возможности:

* создание новых пользователей;
* управление правами доступа;
* предоставление или ограничение доступа к объектам и данным;
* управление учетными записями пользователей.

Такая система позволит контролировать доступ к данным и обеспечить безопасность работы приложения.

### 5. Сохранение данных
Выборки данных, которые делает пользователь, можно сохранить в разных форматах (csv, ...).
Графики, которые делает пользователь, можно сохранить в разных форматах (jpeg, ...).
Графики, которые по умолчанию строятся в дашборде можно сохранить в разных форматах (jpeg, ...).

## Предполагаемая архитектура системы

Система будет состоять из следующих компонентов:

* **Frontend** — веб-интерфейс для визуализации данных и взаимодействия с пользователем.
* **Backend** — серверная часть приложения, обеспечивающая обработку запросов и взаимодействие с базой данных.
* **Database** — база данных, в которой хранятся данные датчиков и информация об объектах (postgreSQL).

## Возможности для дальнейшего развития

В будущем система может быть расширена следующими возможностями:

* поддержка нескольких объектов;
* добавление новых типов визуализаций;

## Цель проекта

Создать удобную систему для мониторинга энергопотребления, которая позволит пользователям получать наглядную информацию о потреблении энергии на объектах.
